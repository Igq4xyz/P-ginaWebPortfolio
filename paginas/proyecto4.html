Proyecto 4: Configuración avanzada de red y servicios

Jupyter Server, doble interfaz de red y optimización en Raspberry Pi 4

Descripción general

Durante las últimas semanas he realizado la configuración y optimización de una Raspberry Pi 4 (6 GB RAM) utilizando Ubuntu Server 24.04 como sistema base. Ya que estoy más familiarizado con este SO para los servidores.

Implementación de doble interfaz de red y resolución de conflictos

Uno de los objetivos principales del proyecto era habilitar una segunda interfaz de red adicional a la integrada en la Raspberry Pi. Durante la configuración inicial de Netplan surgió un error persistente:

brcmfmac brcmf_set_channel set chanspec

La solución consistió en instalar el paquete iproute2 y definir manualmente nuevas tablas de enrutamiento personalizadas en /etc/iproute2/rt_tables, añadiendo las siguientes entradas:

60 60
61 61

Tras aplicar estos cambios, Netplan dejó de producir errores y ambas interfaces comenzaron a funcionar correctamente.

Despliegue de Jupyter Server

Una vez estabilizada la conectividad, configuré Jupyter Server como entorno web interactivo para ejecutar código, tomar notas técnicas y almacenar scripts de automatización.

El acceso se realiza mediante token o contraseña, consultable con:

cat /home/igq/.jupyter/jupyter_server_config.json

Migración de Nextcloud a almacenamiento externo

A la hora de levantar este proyecto debía hacerlo de manera manual diariamente con scrips y ejecutar el típico ./script.sh para que se levante, así que se me ocurió utilizar docker para migrar el Nextcloud y Jupyter.

Como parte de la transición hacia un entorno más autosuficiente, incorporé un dispositivo de almacenamiento externo temporal para proceder a la migración de Nextcloud desde Docker a la Raspberry Pi, mejorando la estabilidad y fiabilidad del servicio.

Así creando un servicio de systemd ya podía levantar todo automático.
