Proyecto 6

Implementación de VaultWarden en el servidor con Docker, dominio y servidor proxy

Descripción

He lanzado VaultWarden en una Raspberry Pi como solución de gestión de contraseñas privadas y alojadas sin depender de plataformas externas. El proyecto está basado en la implementación mantenida por Dani García y está optimizado para arquitecturas ARM.

Requisitos previos

Dominio o subdominio dinámico

Para este proyecto utilicé DuckDNS como proveedor DNS, creando un subdominio para asociarlo al servicio.

A parte de esto me asegur'e de los puertos libres de mi servidor y de ver como funcionaba este proyecto leyendo la documentación oficial.

Documentación oficial de Bitwarden para auto almacenamiento

Reverse Proxy

VaultWarden requiere HTTPS que en comparacion con HTTP este garantiza la transmisión cifrada de datos.Pero no tenia ningun certificado ni tampoco sabia como hacerlo. Para ello implementé un proxy mediante Nginx Proxy Manager ejecutado en Docker. Este proxy gestiona automáticamente certificados SSL con Let's Encrypt y permite publicar el servicio de forma segura.

Despliegue

Se crearon los contenedores Docker correspondientes a:

  • VaultWarden
  • Nginx Proxy Manager (como proxy inverso)

    • Durante la configuración inicial surgieron conflictos de puertos debido a servicios previamente levantados en el sistema, que fueron reasignados correctamente.

    Una vez configurado el reverse proxy y vinculados los certificados SSL al dominio de DuckDNS, el servicio quedó accesible vía HTTPS.

    Resultado

    VaultWarden se ejecuta de forma estable en la Raspberry Pi, protegido mediante cifrado TLS y accesible a través de un dominio externo. El sistema proporciona una solución local, segura y autosuficiente para la gestión de contraseñas sin depender de plataformas externas.

    A parte tengo copias de seguridad de los datos semanalmente, una completa y otra incremental para que los nuevos datos se vayan agregando a los datos anteriores.
