Proyecto 6

Implementación de VaultWarden en Raspberry Pi con Docker, dominio y reverse proxy

Descripción

He desplegado VaultWarden en una Raspberry Pi como solución de gestión de contraseñas autoservida. El proyecto está basado en la implementación mantenida por Dani García y está optimizado para arquitecturas ARM.

Requisitos previos

Dominio o subdominio dinámico

Para este despliegue utilicé DuckDNS como proveedor DDNS, creando un subdominio para asociarlo al servicio.

Reverse Proxy

VaultWarden requiere HTTPS para garantizar la transmisión cifrada de datos. Para ello implementé un reverse proxy mediante Nginx Proxy Manager (NPM) ejecutado en Docker. Este proxy gestiona automáticamente certificados SSL con Let's Encrypt y permite publicar el servicio de forma segura.

Despliegue

Se crearon los contenedores Docker correspondientes a:

  • VaultWarden
  • Nginx Proxy Manager (como proxy inverso)

    • Durante la configuración inicial surgieron conflictos de puertos debido a servicios previamente levantados en el sistema, que fueron reasignados correctamente.

    Una vez configurado el reverse proxy y vinculados los certificados SSL al dominio de DuckDNS, el servicio quedó accesible vía HTTPS.

    Resultado

    VaultWarden se ejecuta de forma estable en la Raspberry Pi, protegido mediante cifrado TLS y accesible a través de un dominio externo. El sistema proporciona una solución local, segura y autosuficiente para la gestión de contraseñas sin depender de plataformas externas.
