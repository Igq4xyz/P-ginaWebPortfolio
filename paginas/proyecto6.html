Proyecto 6 - Implementación de VaultWarden en el servidor con Docker, dominio y servidor proxy

Descripción

He lanzado VaultWarden en una Raspberry Pi como solución de gestión de contraseñas privadas y alojadas sin depender de plataformas externas. El proyecto está basado en la implementación mantenida por Dani García y está optimizado para arquitecturas ARM.

Requisitos previos

Dominio o subdominio dinámico

Para este proyecto utilicé DuckDNS como proveedor DNS, creando un subdominio para asociarlo al servicio.

Me aseguré de tener los puertos libres en el servidor y comprendí el funcionamiento del sistema mediante la documentación oficial.

Documentación oficial de Bitwarden para auto almacenamiento

Reverse Proxy

VaultWarden requiere HTTPS para garantizar la transmisión cifrada de datos. Al no disponer inicialmente de certificados, implementé un proxy inverso mediante Nginx Proxy Manager ejecutado en Docker. Este proxy gestiona automáticamente certificados SSL con Let's Encrypt y permite publicar el servicio de forma segura.

Despliegue

Creé los contenedores Docker:

  • VaultWarden
  • Nginx Proxy Manager (proxy inverso)

Durante la configuración inicial surgieron conflictos de puertos debido a servicios previamente levantados en el sistema, los cuales fueron reasignados correctamente.

Una vez configurado el reverse proxy y vinculados los certificados SSL al dominio de DuckDNS, el servicio quedó accesible vía HTTPS.

Resultado

VaultWarden se ejecuta de forma estable en la Raspberry Pi, protegido mediante cifrado TLS y accesible a través de un dominio externo. El sistema proporciona una solución local, segura y autosuficiente para la gestión de contraseñas sin depender de plataformas externas.

Además, se realizan copias de seguridad semanales: una completa y otra incremental para conservar el histórico de datos.
